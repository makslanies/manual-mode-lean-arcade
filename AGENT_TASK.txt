5. UI / UX — REQUIREMENTS AND ANTI-GLITCH CHECKLIST
================================================================================

5.1. Two layers (required)
  • Canvas — world, gameplay HUD, particles, phone, dispatcher.
  • React/DOM — fullscreen overlays: title / supply / rules / end + #hint.
  Overlay above canvas (higher z-index), dimmed backdrop + blur.
  While an overlay is open, canvas MUST NOT receive clicks
  (pointer-events: none on canvas OR overlay covers inset:0).

5.2. Visual system (GDD §9 + as-built §5)
  Normal=blue/azure · Warning=amber · Critical=red ·
  Improved=green · Auto=cyan · Unobserved=grey fog.
  HUD: at most five primary metrics.
  All 7 zones readable without map scrolling on target resolutions.

5.3. Anti-glitch checklist (required acceptance)

Layout / responsive:
  [ ] No text/button/panel overlaps at 1366×768 and 1920×1080 (AC-09)
  [ ] Also check 1280×720 and ~900×700: nothing clipped; critical buttons clickable
  [ ] Window resize recomputes canvas size + camera; no dead black bars / blurry stretch
  [ ] DPR / devicePixelRatio: sharp text/lines (cap 2); click coords match draw
      (common bug: clicks miss zones)
  [ ] Supply/rules cards do not jump: wrapping long strings; button stable on n/3
  [ ] Hint pill does not cover overlay buttons and does not capture clicks
      (pointer-events: none)
  [ ] Touch: buttons ≥ ~44px; no accidental page zoom (viewport / touch-action)

Layers and input:
  [ ] In title/supply/rules/end, clicks do NOT fall through to canvas
      (director must not walk "under the menu")
  [ ] Closing an overlay leaves no invisible pointer-events blocker
  [ ] Keys do not stick after Alt-Tab / blur / visibilitychange
  [ ] Repeat click on a zone while the director is acting is ignored
  [ ] Space/Enter in an overlay do not trigger world actions; Enter may
      confirm the overlay button — never both
  [ ] Focus: after shift start, focus is not trapped in a hidden button

FSM / timers UI:
  [ ] Pause timers keep ticking in a background tab (prototype watchdog interval)
  [ ] "MOUNT" disabled until selection count == 3; auto-mount stays consistent
  [ ] Rules without sensors unavailable / stub text; no orphan rules
  [ ] play→supply→play2 resets zones for fair comparison (value, failed,
      powerOut, deadlines) per as-built
  [ ] "PLAY AGAIN" clears particles, floats, quips, shake, hitstop

Render / perf:
  [ ] Stable isometric Z-order (no director popping over distant buildings)
  [ ] Particle cap; no float/pulse leaks across shifts
  [ ] Hitstop/shake never permanently break hit-testing
  [ ] FPS drops must not break phase timers (dt clamp; sim ≠ raw rAF)
  [ ] Target 60 FPS on a mid laptop, min 30 FPS; first interactive < 3 s

Accessibility (chapter-0 minimum):
  [ ] Mouse + keyboard work (AC-08)
  [ ] Alerts are not color-only: icon/badge/pulse/sound
  [ ] Master mute (UI or M key); audio errors must not stop the game
  [ ] Optional reduce-flash mode (disable shake/heavy flash)

Copy:
  [ ] Player strings via content dictionaries; no truncated titles on 1366
  [ ] Dynamic results headline when p3 < p1 vs fallback
  [ ] hintOnce tips do not spam every frame or stack on each other
